🚀1. Initiation & Planning

Generate a Project Charter
AI as Project Analyst — Help PMOs rapidly define project scope, governance, and stakeholder alignment
Specific Heavy ChatGPT · Claude
▾

INPUT

  • Project background & business objectives
  • Scope (what's in / what's out)
  • Key stakeholders, sponsor & PM
  • Budget, timeline, and constraints
💡 Even a rough brief works — AI will surface gaps and list assumptions.

OUTPUT

  • Formal project charter (ready to review)
  • Scope, objectives & success criteria
  • Stakeholder RACI matrix draft
  • Risks, assumptions & constraints

When to use

At project initiation — paste your project brief directly. No documents required for the template version.

Template — replace the [ ] parts with your project details
You are a [role, e.g., Senior PMO Specialist]. Create a formal Project Charter for [Project Name]. A [project type] to [objective] for [organization]. Context: Sponsor: [Name]. Stakeholders: [list]. Budget: [approx.]. Go-live: [date]. Constraints: [e.g., regulatory deadline]. Scope (In): [what is included]. Scope (Out): [what is excluded]. Tone: [Executive-level / Team-facing]. Output: Full Charter: Objectives | Scope In/Out | RACI | Milestones | Budget Summary | Risks & Assumptions | Success Criteria | Sign-off section. Evaluation: Flag any section where data is missing. List all assumptions. Highlight top 3 risks to delivery.
Full example — ready to use, attach 2–3 project documents
You are a Senior PMO Specialist with 10+ years of IT transformation experience. Create a formal Project Charter for PMO Service for CBS project. Context: Sponsor: CFO | Stakeholders: IT. Attach: Proposal. Scope (In): PMO Service. Flag any information gap. Output: Full Charter: Objectives | Scope In/Out | RACI | Milestones | Budget | Risks & Assumptions | Success Criteria | Sign-off table. Evaluation: Verify all sections complete. Flag assumptions without a source. Highlight top 3 risks to Go-Live.
Generate a Project Schedule
AI as Project Analyst — Help PMOs structure, prioritize, and track project activities faster
Specific Heavy ChatGPT · Claude
▾

INPUT

  • Project charter (objectives, scope, phases)
  • Team availability / capacity data
  • WBS or deliverable list (optional)
💡 Include phase durations and known dependencies — AI builds a more realistic critical path.

OUTPUT

  • Structured schedule with phases & milestones
  • Task list with owners, durations & dependencies
  • Critical path highlights & schedule risks

When to use

At project kickoff or phase start. Attach charter + team roster.

Template
You are a [role, e.g., Senior PMO Specialist]. Create a structured project schedule for [Project Name] covering phases: [list phases]. Attach: [project charter + team roster or WBS]. Team capacity: [X members, Y hrs/week]. Flag critical path tasks. [Director-level / Team-facing] tone. Output: Phase timeline table (Phase | Start | End | Duration) Task list (Task | Owner | Duration | Dependencies) Critical path with top 3 schedule risks Evaluation: Verify all phases from the charter are covered. Flag tasks without owners or missing dependencies.
Full example
You are a Senior PMO Specialist with 10+ years of IT project delivery experience. Create a structured project schedule for the ERP Integration project covering phases: Discovery, Design, Build, UAT, and Go-Live. The attached deck is the TCP DM&A Phase 3 Kick-off presentation. Use it as the sole source of truth for scope, workstreams, team structure, and timelines. Inputs provided: Phase 3 Kick-off deck (attached). Team availability: all members at 40 hrs/week unless stated otherwise. Output: Phase timeline table (Phase | Start | End | Duration) Task list (Task | Owner | Duration | Dependencies) Critical path with top 3 schedule risks Evaluation: Verify all phases from the charter are covered. Flag tasks without owners or missing dependencies.
Resource Allocation Plan
AI as Resource Planner — Optimize team allocation and forecast capacity gaps before they impact delivery
Specific Heavy ChatGPT · Claude
▾

INPUT

  • Project plan (phases & tasks with durations)
  • Team roster with roles & availability (%)
  • Skills matrix (optional but recommended)
💡 Include actual availability %, not just FTE counts — AI flags over-allocation accurately.

OUTPUT

  • Resource allocation matrix (who does what, when)
  • Over-allocation warnings with recommended fixes
  • Skill gap analysis and hiring/contracting flags

When to use

At project start or when team composition changes. Attach project plan + team roster.

Template
You are a [role, e.g., Senior PMO Resource Manager]. Create a resource allocation plan for [Project Name] covering [timeframe]. Attach: [project plan + team roster with availability % + skills matrix (optional)]. Flag over-allocation above [X]%. Identify skill gaps. [Director-level / PM-facing] tone. Output: Resource allocation matrix (Role | Person | Phase | % Allocated | Notes) Over-allocation flags with recommended mitigations Skill gap summary with hire/contract recommendations Evaluation: Verify total allocation per person does not exceed 100%. Flag phases with no assigned resource.
Full example
You are a Senior PMO. Create a resource allocation plan for Super App project covering from 21 Nov–4 Jun. Attach: the detailed project plan and the team roster (8 resources, varying availability). Flag over-allocation above 90%. Identify skill gaps for UAT phase. PM-facing, concise tone. Output: Resource allocation matrix (Role | Person | Phase | % Allocated | Notes) Over-allocation flags with recommended mitigations Skill gap summary with hire/contract recommendations Evaluation: Verify total allocation per person does not exceed 100%. Flag phases with no assigned resource.
Capacity Forecast
AI as Resource Planner — Optimize team allocation and forecast capacity gaps before they impact delivery
Specific Heavy ChatGPT · Claude
▾

INPUT

  • Resource allocation plan (current)
  • Sprint backlog with task estimates
  • Team calendar / actuals (if available)
💡 Include actuals from past sprints — AI forecasts more accurately with historical velocity.

OUTPUT

  • Capacity forecast: available hours vs. planned per person
  • Burnout/under-utilization risk flags (>90% or <60%)
  • 3 rebalancing recommendations

When to use

Before sprint/phase planning. Attach resource plan + sprint backlog.

Template
You are a [role, e.g., PMO Capacity Planner]. Forecast team capacity for the next [N] sprints for [Project Name]. Identify utilization risks. Attach: [resource allocation plan + sprint backlog with estimates + team calendar (optional)]. Team: [X members, Y hrs/week]. Flag burnout risk above [90]% and under-utilization below [60]%. Concise tone. Output: Capacity table: Sprint | Member | Available Hrs | Planned Hrs | Utilization % Burnout and under-utilization risk flags 3 rebalancing recommendations Evaluation: Verify capacity totals match team availability. Flag any sprint with >2 members at risk.
Full example
You are an experienced PMO Capacity Planner specializing in Agile delivery. Forecast team capacity for Sprints 4–6 for the Super App project. Attach: the resource plan and the Sprint 4–6 backlog with estimates (6 developers, 40 hrs/week). Flag burnout >90% and under-utilization <60%. Concise, actionable format. Output: Capacity table: Sprint | Member | Available Hrs | Planned Hrs | Utilization % Burnout and under-utilization risk flags 3 rebalancing recommendations Evaluation: Verify capacity totals match team availability. Flag any sprint with >2 members at risk.

Generate RAID Log from Project Documents
AI as RAID Analyst — Identify, structure, and maintain Risks, Assumptions, Issues & Dependencies
Specific Heavy ChatGPT · Claude
▾

INPUT

  • Project plan & milestone tracker
  • Meeting notes / MoM documents (last 2–4 weeks)
  • Latest status report
💡 The more meeting notes you attach, the more issues and dependencies AI will surface.

OUTPUT

  • Structured RAID log (Risks, Assumptions, Issues, Dependencies)
  • Severity and priority rating for each item
  • Recommended owners and mitigation actions

When to use

At project start or after major milestones. Attach project plan + meeting notes + status report.

Template
You are a [role, e.g., Senior PMO Risk Analyst]. Review the attached documents for [Project Name] and generate a complete RAID log. Attach: [project plan + meeting notes (last 2–4 weeks) + latest status report]. Categorize each item as Risk/Assumption/Issue/Dependency. Rate severity (H/M/L). Assign recommended owners. Factual tone. Output: RAID table — ID | Category | Description | Severity | Owner | Mitigation/Action | Status Evaluation: Verify each item references a source document. Flag items with no clear owner or mitigation.
Full example
You are a Senior PMO Risk Analyst with expertise in RAID management for IT projects. Review the attached project documents for the Super App project and generate a complete RAID log. Attach: the project plan, MoM from Weeks 15–18, and the Week 18 status report. Categorize each item as Risk/Assumption/Issue/Dependency. Rate severity (H/M/L). Assign recommended owners. Factual tone. Output: RAID table — ID | Category | Description | Severity | Owner | Mitigation | Status Evaluation: Verify each item references a source document. Flag items with no clear owner or mitigation.

Initial Meeting Request Email
AI as Communication Assistant — Craft professional, context-aware meeting emails in minutes
Generic Light ChatGPT · Claude · Gemini · Copilot
▾

INPUT

  • Meeting purpose & key objectives
  • Attendee names, roles & organization
  • 2–3 proposed time slots & platform (Teams/Zoom/in-person)
💡 Provide actual date/time options with timezone — "next week" produces generic outputs.

OUTPUT

  • Professional meeting request email (ready to send)
  • Clear agenda aligned to objectives
  • Polite, formal tone — ready to send

When to use

When scheduling a first-time or formal meeting. No documents needed — works with text input.

Template
You are a [role, e.g., PMO Communication Lead]. Draft a formal meeting request email to [recipient name & role] at [organization] for [meeting purpose]. Objectives: [list 2–3 objectives]. Proposed slots: [option 1], [option 2], [option 3]. Platform: [Teams/Zoom/in-person]. [Formal] tone. Duration: [X mins]. Include a brief agenda. Output: Complete email with subject line, body, and proposed agenda. Evaluation: Confirm subject line is specific, agenda items match objectives, and tone is appropriate for [recipient level].
Full example
You are a professional PMO Communication Lead at Bluebik Group. Draft a formal meeting request email to the Head of Finance and IT Director at ABC Corporation for the ERP Integration project kickoff. Objectives: align on project scope, confirm resource commitments, agree on governance structure. Slots: Mon 23 Jun 10:00–11:00 BKK / Tue 24 Jun 14:00–15:00 / Wed 25 Jun 09:00–10:00. Platform: Microsoft Teams. Formal, professional tone. Duration: 60 mins. Include brief agenda. Output: Complete email with subject line, body, and proposed agenda. Evaluation: Subject line must name the project. Agenda must cover all 3 objectives. Tone appropriate for C-Suite.

New proposed slots: Wed 19 Jun 14:00–15:30, Thu 20 Jun 10:00–11:30, Mon 24 Jun 14:00–15:30. Same attendees. Polite, professional tone. Reference original date. Max 3 paragraphs. Output: Complete email with subject line and body. Evaluation: Confirm original date referenced, all 3 new slots listed, and CFO travel not detailed (out of respect).

Attach: project schedule + resource plan. Assume 1-week delay. Show best-case, likely-case, worst-case outcomes. Director-level tone. Output: Impact summary per scenario (Go-Live date shift, affected tasks) Revised critical path Top 3 mitigation options with effort and risk tradeoffs Evaluation: Confirm the critical path recalculation is consistent with the schedule. ▾

When to use

After a CR is submitted — before it is approved, to quantify the full impact.

Attach: [CR form + current project plan + scope baseline]. Note the critical path and any fixed go-live dates. Factual, evidence-based tone. Output: Impact summary per dimension: Scope | Schedule | Budget | Resource | Risk (H/M/L) Overall change risk rating with rationale Approval recommendation with conditions or caveats Evaluation: Confirm impact estimates are sourced from attached documents. Flag any dimension without a quantified estimate.
Full example
You are a Senior PMO Business Analyst with expertise in change impact evaluation. Perform a Change Impact Assessment on Super App project. Assess: scope, schedule, budget, resource, and risk impact. The attached CR document relates to the Thailand Post Super Application project delivered by Bluebik Group PCL. The project critical path runs sequentially through Design → Build → Test → Deploy. All impact estimates must be traceable to the attached CR or project Timeline sheet. Do not infer, assume, or estimate any figure not present in the source documents. Evidence-based tone. Output: Impact summary per dimension | Overall risk rating | Approval recommendation Evaluation: Verify all impact estimates reference the attached plan. Flag anything unquantified.
